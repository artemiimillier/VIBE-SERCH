## КРИТИЧЕСКИЕ ИСПРАВЛЕНИЯ К ПЛАНУ

В плане есть ошибка, которую нужно исправить при реализации Фазы 2:

**Верификация в 1 вызов НЕ РАБОТАЕТ.** Исследование ICLR 2024 (Huang et al.) доказало: LLM не может объективно критиковать свой же вывод в одном контексте (sycophancy bias). Поэтому вместо одного вызова Sonnet с "двойной ролью" (адвокат + критик) нужно делать **2 отдельных вызова с информационной асимметрией**:

```
Вызов 1 - Аналитик (Sonnet):
"Вот факт и его источник. Оцени: насколько это важно, насколько это достоверно,
какие аргументы ЗА. Ответь JSON: {significance, credibility_args, key_claims}"

Вызов 2 - Верификатор (Sonnet):
"Вот набор утверждений (key_claims). БЕЗ контекста первого анализа.
Найди слабости: что может быть преувеличено, неточно, вырвано из контекста.
Ответь JSON: {weaknesses, hype_risk, confidence, hype_rating, reasoning}"
```

Ключевой принцип: верификатор НЕ видит рассуждения аналитика - только голые утверждения. Это создает информационную асимметрию, которая ломает sycophancy bias.

Стоимость: x2 от плана на верификацию (~$1 вместо $0.50). Но это ЯДРО продукта - здесь экономить нельзя.

## Правила для субагентов

1. **Код на английском, комментарии на английском, общение на русском**
2. **Type hints на всех функциях, Pydantic для валидации**
3. **Все ключи из .env, никогда хардкод** - если в коде есть строка похожая на ключ, это баг
4. **Функции не длиннее 30 строк**
5. **Никаких лишних зависимостей** - только то, что в плане
6. **git add поименно** - никогда `git add .`
7. **Не пушить и не коммитить** без явной команды пользователя
8. **.env не читать целиком** - только через Pydantic Settings
9. **Rate limits**: retry с backoff на всех API-вызовах
10. **Логирование**: каждый этап пайплайна логирует время и стоимость

## После завершения ВСЕХ фаз (0-4)

### Финальная проверка (3 субагента последовательно):

**Субагент 1: Финальная проверка на ошибки**
- Прочитай ВСЕ файлы в src/
- Найди: баги, несоответствия между модулями, отсутствующие edge cases
- Проверь: все import-ы работают, все модели совместимы между модулями
- Запусти полный пайплайн: `python -m src.main` (если нет API ключей - dry run)
- Запусти `ruff check src/` и `pytest`
- Исправь все найденные проблемы

**Субагент 2: Security review**
- Прочитай ВСЕ файлы в src/
- Проверь:
  - Нет хардкода секретов (API ключи, токены)
  - .env не читается напрямую (только через Pydantic Settings)
  - Нет SQL/command injection (если есть shell-вызовы или запросы)
  - Input validation на всех внешних данных (Reddit, LLM output)
  - Нет утечки данных в логах (ключи, токены не логируются)
  - Telegram bot token защищён
  - Нет open redirects, SSRF через URL из Reddit
- Исправь все найденные проблемы

**Субагент 3: Финальный smoke test**
- Запусти `ruff check src/ && ruff format --check src/`
- Запусти `pytest -v`
- Попробуй `python -c "from src.main import main"` - должен импортироваться
- Попробуй `python -c "from src.config import settings"` - должен работать
- Проверь что все файлы из структуры плана созданы
- Проверь что .env.example содержит все нужные переменные
