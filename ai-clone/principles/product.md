# Principles - убеждения про продукт

## Продукт

- Качество > количество: лучше 5 проверенных фактов, чем 50 непроверенных
- Каждый факт проходит мультиагентную верификацию
- Ранжирование по доверию: первоисточники > агрегаторы > инфоблогеры
- Каждая цифра - со ссылкой, каждое утверждение - с примером
- Если источник сомнительный - не публикуем, а не публикуем с оговоркой
- Пользователь получает суть за 5 минут, не тратит время на скролл

## Код

- Простое решение побеждает элегантное
- Type hints везде, Pydantic для валидации
- Каждый сканер - отдельный модуль с единым интерфейсом
- Функции не длиннее 30 строк, разбивать если длиннее
- One task - one chat: не смешиваем задачи в одной сессии
- План в файле (plans/), не в голове Claude

## Бизнес

- Монетизация через подписку, не рекламу: пользователь платит за качество, не за клики
- Рост через качество контента, не маркетинг: если дайджест полезен - сарафан работает сам
- Сначала один канал доставки, потом масштаб: MVP = один канал, потом расширяем
- Честность с пользователем: не преувеличиваем возможности, не скрываем ограничения
- Скорость итераций: быстрее выпустить, получить фидбэк, улучшить
