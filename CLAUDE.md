# VIBE-SERCH

Ежедневный AI-сканер мира вайб-кодинга и нейросетей.
Сканирует источники, верифицирует факты мультиагентной системой, формирует утренний дайджест для разработчиков и предпринимателей.

## Порядок чтения в начале сессии

1. `ai-clone/INDEX.md` - кто думает (стиль, принципы, голос)
2. `business/INDEX.md` - о чем (продукт, аудитория, цели)
3. Этот файл - правила репо
4. Зона конкретной задачи (код, план, ретроспектива)

## Роутинг: вопрос -> файл

| Зачем мы это делаем | `business/company/mission.md` |

| Каналы привлечения, контент-стратегия | `business/marketing/channels.md` |

## Правила репо

### Безопасность
- Все ключи и токены только через `.env`, никогда хардкод
- Никогда не показывать значения секретов в чате
- `.env` не читать целиком - только `grep` нужной переменной
- Перед push: `pytest` и `ruff check`
- Не пушить и не коммитить без явной команды

### Код
- Type hints на всех функциях, Pydantic для валидации данных
- Каждый сканер - отдельный модуль в `src/scanners/` с единым интерфейсом
- Rate limits: уважать лимиты API источников, использовать backoff
- Функции не длиннее 30 строк, разбивать если длиннее

### Git
- `git add` поименно, никогда `git add .` и `git add -A`
- Перед push: `git log origin/main..main` - проверить что уходит
- Коммит-сообщения: глагол + суть, на английском

### Язык
- Общение на русском
- Код и комментарии на английском
- Только дефис, никогда длинное тире

## Триггеры -> действия

| Фраза пользователя | Что делать |
|---|---|
| "Сделай план" | Создай `plans/YYYY-MM-DD_название.md` (цель, шаги, критерии, риски) |
| "Закончили фичу" | Цикл проверки (см. ниже) |
| "Конец сессии" | Рефлексия (см. ниже) |
| Работа с текстом/UI | Сначала прочитай `ai-clone/voice/tone.md` и `business/audience/avatar.md` |
| Бизнес-решение | Сначала прочитай `business/goals/current.md` и `business/economics/model.md` |

## Цикл после фичи

1. Найди ошибки - докажи примером
2. Граничные случаи - что если пустой ответ, таймаут, невалидные данные
3. Сверка с бизнес-контекстом - `business/audience/avatar.md`
4. Security review - нет ли утечки ключей, инъекций, открытых эндпоинтов

## Самообучение

В конце каждой сессии:

1. Проверь: узнал ли что-то новое о проекте или пользователе?
2. Проверь: какие правила нарушил агент?
3. Если да - создай файл в `ai-clone/feedback/` по шаблону `_TEMPLATE.md`
4. Запиши ретроспективу в `retrospectives/YYYY-MM-DD_название.md`:
   - Задача, подход, результат (да/нет/частично)
   - Что можно лучше, что узнал нового
